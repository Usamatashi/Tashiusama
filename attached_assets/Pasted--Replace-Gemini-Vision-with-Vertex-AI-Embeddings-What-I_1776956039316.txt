What Gets ADDED
1. Embedding generator helper — artifacts/api-server/src/lib/embeddings.ts (new file, ~50 lines)

Wraps Vertex AI PredictionServiceClient.predict()
Takes an image buffer or URL, returns number[] (1408 floats)
Handles auth via existing service account
Uses dimension: 1408 for max accuracy (or 512 if we want smaller storage)
2. Auto-embed on product create/update — modify the existing admin product endpoints

When admin uploads/updates a product diagram → fetch the diagram image → generate embedding → store in Firestore as diagramEmbedding: [1408 floats]
Runs once per product, not per scan
3. One-time backfill endpoint — POST /admin/backfill-embeddings (admin-only, requires auth)

Loops through all existing products that have diagramUrl but no diagramEmbedding
Generates embeddings for each
Returns count of products processed
You call it once after deploy to populate embeddings for your existing catalog
4. New /identify route logic:

1. Receive photoBase64 from mobile app
2. Generate embedding for the photo (~300ms)
3. Pull all products with diagramEmbedding from Firestore
4. Compute cosine similarity against each
5. Sort by score, take the top match
6. If score ≥ 0.75 → return matched product + confidence label
   If 0.65–0.75 → return product but flag as "low confidence"  
   If < 0.65 → return "no confident match, please retake"
7. Total time: ~1 second

5. Mobile-side change — artifacts/tashi/app/(user)/products.tsx (or wherever the camera is)

Resize the captured photo to max 1024px before uploading using expo-image-manipulator
Reduces upload size from ~3MB to ~150KB
Faster scans, lower bandwidth, better embedding quality
