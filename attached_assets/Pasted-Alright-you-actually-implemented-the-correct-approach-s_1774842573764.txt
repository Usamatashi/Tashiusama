# 🚨 The 3 REAL reasons this still happens

## 🔴 1. Permissions are `undefined` at first render (MOST COMMON)

Expo Router builds tabs **on first render**.

If this happens:

```js
settings = undefined // initially
```

Then:

```js
tabHref("tab_payments") => undefined
```

👉 Which means:
**tab is allowed initially → router keeps it**

Even if settings update later, the tab **won’t disappear**.

---

### ✅ Fix (CRITICAL)

Block rendering until permissions are loaded:

```js
if (!settings) {
  return null; // or loader
}
```

OR safer:

```js
const isReady = settings !== null && settings !== undefined;

if (!isReady) return null;
```
